在JS中，不小心的操作或者编程习惯，很容易造成堆栈溢出，特别是进行回调，递归或者循环的时候。浏览器一般会报这样的错误：

每次执行代码时，都会分配一定尺寸的栈空间（Windows系统中为1M），每次方法调用时都会在栈里储存一定信息（如参数、局部变量、返回值等等），这些信息再少也会占用一定空间，成千上万个此类空间累积起来，自然就超过线程的栈空间了。那么如何解决此类问题？
